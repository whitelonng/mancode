mancode/Documentation

Practice to playoffs

Give your coding agent
a real playbook.

mancode is a local-first workflow harness for the coding agent you already use. It adds project context, task intensity, durable artifacts, and review discipline without replacing Claude Code, Cursor, Codex, Copilot, or ZCode.

Start here

Use solo for day-to-day work. It performs one bounded diff self-check with no extra reviewer loop. Escalate to /mamba for diagnosis and /man when a production change deserves a plan, tests, and risk-based review.

Install & initialize

Install mancode globally, enter a project directory or a new empty folder, then choose the adapter or adapters to initialize. Node.js 20+ is required. Windows CMD, PowerShell, and Git Bash are supported natively; Git itself is optional and only improves team auto-detection. Claude Code hooks run with Node, not Bash or jq.

01Install

Install the CLI once on your machine.

02Enter

Run it from the project you want mancode to understand.

03Initialize

Create local state, scan the project, and install the adapter.

$ npm install -g mancode
$ cd your-project
$ mancode init

Initialize a specific platform

Use --platform when the current project should target a particular coding surface from the start.

$ mancode init --platform codex,cursor

mancode init guides you through one, many, or all agent platforms. In an empty folder it can initialize a generic project—no git init or npm init required. Add Git or a manifest later, then run mancode refresh-project to update project facts and installed static adapters.

OptionUse it whenWhat it does
--forceRepair or reinstall generated configurationReinstalls while preserving scanned style tokens.
--yesCI or a non-interactive shellSkips the generic-project confirmation; pair it with --platform.
--teamYou need shared memory immediatelyForces team mode on.
--no-teamThis is a personal projectForces team mode off.
--style NAMEYou have a named visual preferenceSaves it as the default style preference.

Recommended workflow

mancode is deliberately not a mandatory ceremony layer. Choose the smallest mode that creates enough confidence for the task.

Routine change

solo

Use for a small feature, refactor, or UI tweak. You get project context, the narrowest meaningful validation, and one diff-only self-check without another reviewer.

Bug or uncertain behavior

/mamba

Use when the right answer depends on reproduction and real validation—not a plausible patch.

Production or high-risk change

/man

Use when planning, verification, and independent review need to be durable and visible.

Team or maintenance

/manteam · /manps

Use shared memory for a team; use health scans before a cleanup or dependency-maintenance pass.

Rule of thumb

Do not use /man just because a task is long. Use it when a wrong decision would be expensive, hard to reverse, or difficult to review after the fact.

Modes

The five working modes cover different kinds of engineering pressure. /mansolo is the reset command that returns the project to the default solo mode.

ModeBest forWhat changes
soloDaily codingLightweight hooks, style awareness, YAGNI checks, and one bounded diff self-check.
/mambaDiagnosisReproduce the failure, find the cause, validate real user paths, and run regression checks.
/manHigh-stakes workFull nine-step workflow with a plan gate, targeted/full risk review, and one remediation round.
/manteamTeam projectsShared memory, decision records, coordination, and Conventional Commits guidance.
/manpsMaintenanceDeterministic project-health scan with Markdown and JSON reports.

How to invoke a mode

Claude Code and Cursor expose slash-style commands. Codex in the ChatGPT desktop app, CLI, and IDE reads repository skills, so $man, $mamba, and related mentions are the portable explicit syntax.

# Claude Code / Cursor
/mamba
/man
/manteam

# Codex app / CLI / IDE
$mamba
$man
$manteam

CLI commands

Use these commands from an initialized project unless noted otherwise. Commands that read state fail safely if .mancode/state.json is missing.

mancode init

Start

Creates .mancode/, detects the project profile, scans design tokens when relevant, and installs the selected adapter.

  • Run once for a new project.
  • Use --platform to target a specific agent.
  • Use --force to repair generated files.

mancode status [--json]

Inspect

Shows the current mode, detected stack, installed platforms, and readiness. Claude Code installations also report hook registration and estimated injection size.

  • Use plain output for a quick human check.
  • Use --json for scripts or CI.

mancode install <platform>

Adapter

Installs or repairs one platform adapter after initialization.

  • Platforms: claude-code, cursor, codex, copilot, zcode.
  • Use --force to rewrite generated configuration.
  • Use --minimal for solo-mode essentials only.

mancode list-platforms

Discover

Lists the adapters this version knows how to install. Use this before scripting platform selection.

mancode workflow …

Track

Creates and manages validated metadata for man, mamba, and manteam workflow runs.

  • create <mode> "<task>" starts a record.
  • update <taskId> advances steps or status.
  • review <taskId> … enforces required domains, blockers, and one remediation round.
  • list, show, and clean inspect and maintain records.

mancode manps [area]

Scan

Runs a deterministic preseason health scan. Areas include deps, security, dead-code, and config.

  • Writes Markdown and JSON reports under .mancode/.
  • Scans only; remediation remains an explicit decision.

mancode refresh-style

Rescan

Refreshes the project profile and, when UI assets exist, rescans design tokens.

  • Updates project-profile.json and style-tokens.json.
  • Afterward, run mancode install <platform> --force for static-instruction adapters.

mancode version

Version

Prints the installed CLI version. Use it when reporting an issue or validating an update.

Workflow records

mancode workflow manages metadata used by the strict modes. The records make progress, plan versions, skipped steps, and review outcomes inspectable after the chat is over.

$ mancode workflow create man "refactor auth module"
$ mancode workflow update <taskId> --step 4 --plan-version 2
$ mancode workflow review <taskId> init --review-depth full
$ mancode workflow review <taskId> complete --review-domain quality --report film-report-1.md --blockers Q1
$ mancode workflow review <taskId> remediate --resolved Q1
$ mancode workflow create mamba "verify auth regression" --parent-task <taskId>
$ mancode workflow show <taskId> --json
$ mancode workflow clean --older-than 30d --dry-run
Constraint

A linked mamba child may only be created while its parent workflow is active at Step 6. New governed workflows also cannot complete until their required review domains are recorded and all blocker IDs are resolved.

Platforms & adapters

mancode intentionally adapts to each surface instead of pretending every tool has the same extension points. Claude Code receives the fullest implementation; other tools receive persistent rules, skills, or managed instruction files.

PlatformInstalled integrationNotes
Claude CodeHooks, skills, subagents, workflow integrationFull integration: session start and prompt-submit hooks can inject project context.
Cursor.cursor/rules/*.mdcRules supply persistent project guidance and mode descriptions.
CodexAGENTS.md + .agents/skills/Use portable $man* skill mentions in the app, CLI, and IDE.
GitHub Copilot.github/copilot-instructions.mdManaged instruction block; no equivalent hook injection today.

Refresh project style

Run this after a design-system migration, substantial UI change, or when a previously non-UI project gains a user interface.

$ mancode refresh-style
$ mancode install cursor --force
  • Claude Code reads refreshed style information through its hooks.
  • Cursor, Codex, and Copilot use static adapter instructions, so reinstall their adapter with --force after a refresh.
  • Only UI projects receive design-token-specific guidance.

Uninstall safely

Uninstall preserves user-authored rules, instructions, and unrelated Claude settings. You can remove one adapter or remove all mancode artifacts from a project.

Remove one adapter

$ mancode uninstall cursor --force

Clean reinstall

$ mancode uninstall --all --force
$ mancode init
$ mancode install <platform>

Remove mancode completely

$ mancode uninstall --all --force
$ npm uninstall -g mancode
Before you remove it

Review .mancode/ before committing or deleting it. Workflow evidence and browser artifacts can contain project context that your team may want to retain or explicitly ignore.

Troubleshooting

mancode init says the directory is not a project

An empty folder is offered as a generic project in an interactive terminal. Non-empty unrecognized folders stay protected; enter the actual project directory. In scripts, use mancode init --empty --platform <platform> only for an intentionally empty directory.

Claude Code hooks are not running

Restart Claude Code after initialization so it reloads .claude/settings.json. Then run mancode status. If the hooks are still missing, run mancode install claude-code --force.

A platform is “not ready”

The target files are missing or its managed block was changed. Run mancode install <platform> --force to regenerate mancode-owned content while preserving user content outside the markers.

Cursor rules do not trigger

Confirm that .cursor/rules/mancode-*.mdc exists. Context, practice, and solo rules always apply; the higher-intensity rules activate through their descriptions, so invoke the relevant mode explicitly.
