mancode/文档中心

训练到季后赛

给你的编码 Agent
一套真战术。

mancode 是加在既有编码 Agent 之上的本地优先工作流层。它不替代 Claude Code、Cursor、Codex、Copilot 或 ZCode，而是增加项目上下文、任务强度、可追溯产物和审查纪律。

从这里开始

日常任务用 solo：只做一次 diff 自检，不启动额外 reviewer 循环；需要定位根因时切换 /mamba；生产级或高风险改动再使用 /man，获得计划、验证和有界风险审查。

安装与初始化

先全局安装 mancode，进入项目目录或全新的空目录，再选择要初始化的一个或多个适配器。需要 Node.js 20+；原生支持 Windows CMD、PowerShell 和 Git Bash。Git 是可选依赖，只用于增强团队自动检测；Claude Code hooks 直接由 Node 执行，不需要 Bash 或 jq。

01安装

在机器上全局安装 CLI。

02进入项目

让 mancode 在需要理解的项目根目录运行。

03初始化

创建本地状态、扫描项目并安装适配器。

$ npm install -g mancode
$ cd your-project
$ mancode init

指定平台初始化

如果你知道当前项目要面向哪个 Agent，可在初始化时指定平台。

$ mancode init --platform codex,cursor

mancode init 会引导选择一个、多个或全部 Agent 平台。空目录可初始化为通用项目，不需要先运行 git init 或 npm init；以后加入 Git 或 manifest 后执行 mancode refresh-project，同步项目事实和已安装的静态适配器。

选项适用情况作用
--force修复或重装生成的配置重装时保留已扫描的风格 token。
--yesCI 或无交互环境跳过通用项目确认；应同时指定 --platform。
--team立即需要团队记忆强制开启团队模式。
--no-team个人项目强制关闭团队模式。
--style NAME已有命名的审美偏好保存为默认风格偏好。

推荐用法

mancode 的目的不是让所有任务都变重，而是为不同风险选择足够的工程纪律。

日常改动

solo

适合小功能、重构或 UI 微调。使用最窄验证和一次 diff 自检，不调用额外 reviewer。

Bug 或不确定行为

/mamba

当正确答案依赖复现与真实验证，而不是一个看起来合理的补丁时使用。

生产或高风险改动

/man

当计划、测试和有界风险审查需要被记录、复查和交接时使用。

团队或维护

/manteam · /manps

团队共享记忆；维护前先执行健康扫描，避免拍脑袋式清理。

判断原则

不要因为任务很长就自动使用 /man。当错误代价高、难以回滚，或事后难以审查时，再升级到季后赛模式。

模式

五个工作模式对应不同工程压力。/mansolo 是回到默认 solo 模式的重置命令。

模式适合场景会做什么
solo日常编码轻量 hooks、YAGNI、最窄验证和一次受限 diff 自检。
/mamba诊断复现缺陷、定位根因、驱动真实用户路径并回归验证。
/man高风险改动完整九步工作流、计划关卡、定向/完整风险审查和一轮修复。
/manteam团队项目共享记忆、决策记录、协作和 Conventional Commits 指导。
/manps维护输出 Markdown 与 JSON 的确定性项目健康报告。

怎样调用

Claude Code 与 Cursor 使用 slash 风格命令；ChatGPT 桌面端 Codex、Codex CLI 和 IDE 从项目 skill 中读取，因此 $man、$mamba 等是稳定的显式调用方式。

# Claude Code / Cursor
/mamba
/man
/manteam

# Codex app / CLI / IDE
$mamba
$man
$manteam

CLI 命令

以下命令除非另有说明，都应该在已初始化的项目中运行。缺少 .mancode/state.json 时，依赖项目状态的命令会安全失败。

mancode init

开始

创建 .mancode/，检测项目 profile，在存在 UI 时扫描设计 token，并安装所选适配器。

  • 新项目首次使用。
  • --platform 指定 Agent。
  • --force 修复生成文件。

mancode status [--json]

检查

显示当前模式、检测到的技术栈、已安装平台和就绪状态。Claude Code 安装还会显示 hook 注册与预计注入大小。

  • 默认输出用于人工检查。
  • --json 适合脚本或 CI。

mancode install <platform>

适配器

初始化后安装或修复单个平台适配器。

  • 平台：claude-code、cursor、codex、copilot、zcode。
  • --force 重写生成配置。
  • --minimal 只安装 solo 必需项。

mancode list-platforms

发现

列出当前版本可以安装的适配器。自动化选择平台前可以先运行它。

mancode workflow …

记录

为 man、mamba 与 manteam 创建和维护受校验的工作流元数据。

  • create <mode> "<task>" 创建记录。
  • update <taskId> 更新步骤或状态。
  • review <taskId> … 约束审查领域、blocker 和唯一一轮修复。
  • list、show、clean 用于检查和维护。

mancode manps [area]

扫描

运行确定性的季前赛健康检查。可指定 deps、security、dead-code、config。

  • 报告写入 .mancode/。
  • 默认只扫描，整改仍是明确决策。

mancode refresh-style

刷新

刷新项目 profile；检测到 UI 时重新扫描设计 token。

  • 更新 project-profile.json 与 style-tokens.json。
  • 静态 instruction 适配器需要再运行 mancode install <platform> --force。

mancode version

版本

打印当前安装的 CLI 版本。提交 issue 或确认升级时使用。

工作流记录

mancode workflow 管理严格模式使用的元数据。它让进度、计划版本、跳过步骤、审查领域和 blocker 状态在聊天结束后仍可检查。

$ mancode workflow create man "refactor auth module"
$ mancode workflow update <taskId> --step 4 --plan-version 2
$ mancode workflow review <taskId> init --review-depth full
$ mancode workflow review <taskId> complete --review-domain quality --report film-report-1.md --blockers Q1
$ mancode workflow review <taskId> remediate --resolved Q1
$ mancode workflow create mamba "verify auth regression" --parent-task <taskId>
$ mancode workflow show <taskId> --json
$ mancode workflow clean --older-than 30d --dry-run
约束

关联的 mamba 子任务只能在父工作流处于 Step 6 且仍在进行时创建。新的严格 workflow 也必须完成所需审查领域并清零 blocker 后才能完成。

平台与适配器

mancode 不假装所有工具都有相同扩展点。Claude Code 获得最完整的原生能力；其他平台通过持久 rules、skills 或受控 instructions 适配。

平台安装内容说明
Claude CodeHooks、skills、subagents、workflow完整集成：会话启动与提交提示时可注入项目上下文。
Cursor.cursor/rules/*.mdc通过 rules 保存项目指导与模式描述。
CodexAGENTS.md + .agents/skills/在 ChatGPT 桌面端、CLI 与 IDE 使用 $man* 触发 skill。
GitHub Copilot.github/copilot-instructions.md受控 instruction block；当前没有等价 hook 注入。
ZCodeAGENTS.md + 预备项目 skills项目 skill 发现与 slash command 路径仍是发布验证门槛。

刷新项目风格

设计系统迁移、UI 大幅变更，或项目从非 UI 变成有界面时运行。

$ mancode refresh-style
$ mancode install cursor --force
  • Claude Code 通过 hooks 读取刷新后的风格信息。
  • Cursor、Codex 与 Copilot 使用静态适配指令，因此刷新后需要 --force 重装适配器。
  • 只有确认存在 UI 资产的项目才会得到设计 token 相关指导。

安全卸载

卸载会保留用户编写的 rules、instructions 和无关的 Claude settings。你可以移除单个平台适配器，也可以清除项目内的全部 mancode 产物。

移除一个适配器

$ mancode uninstall cursor --force

完整重装

$ mancode uninstall --all --force
$ mancode init
$ mancode install <platform>

完全移除 mancode

$ mancode uninstall --all --force
$ npm uninstall -g mancode
删除前

提交或删除 .mancode/ 前请先检查。工作流证据和浏览器产物可能包含你希望保留或明确忽略的项目上下文。

故障排查

mancode init 提示目录不是项目

请在 git 仓库或带有已识别 manifest 的目录中运行。初始化器不会把任意文件夹当成项目。

Claude Code hooks 没有触发

初始化后重启 Claude Code，让它重新加载 .claude/settings.json。再运行 mancode status；仍缺失时执行 mancode install claude-code --force。

平台状态显示“not ready”

目标文件缺失或受控区被编辑。运行 mancode install <platform> --force 重新生成 mancode 负责的内容，标记外的用户内容会保留。

Cursor rules 没有触发

确认 .cursor/rules/mancode-*.mdc 存在。context、practice、solo 一直生效；高强度规则按描述触发，因此请显式调用对应模式。

没有找到匹配的文档章节。

mancode 本地优先，不发送遥测。
