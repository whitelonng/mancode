03 / 先计划再选择solo 交付或完整治理。

/man 工作流

从需求对齐
到确认计划。

每次规划都会在 .mancode/<namespace>/workflows/<ULID>/ 产生可读记录。计划确认后，可交给 solo 轻量开发，也可继续完整九步治理。

  1. 01球探报告梳理代码、风险与未知项
  2. 02需求澄清确认真正要解决的问题
  3. 03计划输出可验证步骤
  4. 04计划关卡solo、完整治理、仅计划或修改
  5. 05实施完整 /man 才进入
  6. 06验证与范围验证后选择审查深度
  7. 07定向审查有证据的质量 finding
  8. 08风险防守高风险时审查安全边界
  9. 09收尾一轮修复、复验、停止

06 / 跨会话续接换窗口，不丢任务脉络。

一项任务。
下个会话继续。

mancode 把目标、需求、计划、检查结果和交接信息保存在稳定 TaskRef 下。新聊天窗口或 CLI session 可以恢复这项任务，只加载当前目的需要的 Context Pack，不依赖旧窗口一直打开。

步骤mancode 保留什么边界
保存稳定 TaskRef 关联需求、计划、结果和交接信息。持久上下文
恢复mancode context resume <TaskRef> --session <id> --client <client>显式身份
读取按目的生成 Context Pack，恢复真正有用的任务状态。减少噪声
隔离不复制原始聊天记录，不同客户端 session 仍然隔离。边界清晰
